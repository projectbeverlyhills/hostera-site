## 1. Назначение продукта

Hostera Platform — трёхсторонняя B2B2C маркетплейс-платформа в категории premium hospitality. Соединяет три роли в единой экосистеме: **Гость (B2C) → Ресторан (B2B) → Дистрибьютор (Partner)**. Платформа работает как PWA (Progressive Web App): устанавливается на устройство, работает на мобильном и десктопе без App Store.

Текущий ресторанный MVP (`index.html`) остаётся as is — Platform является его эволюцией, а не заменой.

---

## 2. Ключевые принципы

- **Три роли — три интерфейса.** Гость, Ресторан и Дистрибьютор имеют отдельные UI-пространства с общей базой данных.
- **Hostera = платёжный агент.** Все транзакции (B2C и B2B) проходят через платформу. Дистрибьютор получает выплату минус комиссию Hostera.
- **Real-time инвентарь.** Статус наличия позиций виден всем сегментам в реальном времени.
- **Apple-grade UX.** Дизайн-система идентична текущему MVP: hairline outlines, iOS easing, Proxima Nova, light/dark themes.
- **PWA-first.** Нет нативных приложений на первом этапе. Установка на Home Screen через браузер.

## 4. Сегмент 1 — Guest App (B2C PWA)

### 4.1 Авторизация

| Поле | Требование |
|---|---|
| Вход | Email + password, Google OAuth |
| Регистрация | Email, имя, опционально — дата рождения (верификация возраста для алкоголя) |
| Сессия | Persistent login (JWT refresh token), "Remember me" |
| Восстановление | Email-ссылка для сброса пароля |
| Гостевой режим | Просмотр меню без регистрации. Корзина и покупка — только авторизованным |

### 4.2 Главная страница

- Карточки ресторанов экосистемы (фото, название, город, краткое описание)
- Фильтр по городу / типу кухни / наличию wine list
- Featured section — промо от ресторанов / дистрибьюторов (платное размещение)
- Глобальная поисковая строка (см. 4.3)

### 4.3 Кросс-ресторанный поиск вин

**Это центральная функция сегмента.**

- Поиск по: названию вина, производителю, региону, сорту, vintage, стилю
- Результат: список ресторанов, где эта позиция есть **в наличии** (live inventory)
- Статус на карточке: `● В наличии` / `◐ Последние бутылки` / `○ Нет в наличии`
- Карточка результата: фото вина, ресторан, цена по меню, кнопка «Купить онлайн» (если дистрибьютор подключён)
- Переход в меню ресторана из результата поиска
- Сортировка: по близости / по цене / по рейтингу ресторана

### 4.4 Меню ресторана (встроенный просмотр)

- Полный рендер меню ресторана внутри Guest App (переиспользует логику текущего MVP)
- Табы по типам меню: Wine / Bar / Coffee / Pastry
- Карточка позиции: фото, название, описание, цена, аллергены, vintage
- Кнопка «Купить онлайн» на карточке вина (если доступна доставка от дистрибьютора)
- Поиск внутри меню ресторана

### 4.5 Покупка вина (D2C через дистрибьютора)

**User flow:**

```
Карточка позиции
  └→ Кнопка «Купить онлайн»
       └→ Модальное окно: цена дистрибьютора + стоимость доставки + срок
            └→ «Добавить в корзину»
                 └→ Корзина (мульти-позиция от разных дистрибьюторов)
                      └→ Checkout: адрес доставки + способ оплаты
                           └→ Оплата (Hostera как агент)
                                └→ Подтверждение + email-квитанция
                                     └→ Трекинг заказа
```

**Требования к корзине:**
- Позиции от разных дистрибьюторов объединяются в одну корзину
- Если дистрибьюторы разные — отдельные suborders, одна оплата
- Минимальный заказ — на усмотрение дистрибьютора (настраивается в Portal)

**Требования к оплате:**
- Hostera принимает платёж (карта, Apple Pay, Google Pay)
- Автоматический split: дистрибьютору (GMV минус комиссия Hostera)
- Hostera удерживает комиссию (8–12% B2C) до выплаты

**Статусы заказа:**
`Оплачен → Подтверждён дистрибьютором → В сборке → Передан курьеру → Доставлен`

### 4.6 Профиль гостя

- История заказов с возможностью повторного заказа
- Избранные вина (wish list)
- Вкусовой профиль (опционально заполняется: регион, стиль, ценовой диапазон)
- Hostera Pass — управление подпиской ($9.99/мес или $79/год)

### 4.7 Hostera Pass (Premium membership)

| Функция | Free | Pass |
|---|---|---|
| Просмотр меню | ✓ | ✓ |
| Поиск по ресторанам | ✓ | ✓ |
| Покупка вина | ✓ | ✓ |
| Ранний доступ к allocation | — | ✓ |
| Персональные sommelier-рекомендации | — | ✓ |
| Приоритетная доставка | — | ✓ |
| Эксклюзивные curated collections | — | ✓ |

### 4.8 Уведомления (Push / Email)

- Поступление в наличие позиции из wish list
- Статус заказа (каждый этап)
- Персональные рекомендации (не чаще 1 раза в неделю)
- Allocation-уведомления для Pass-подписчиков

## 5. Сегмент 2 — Restaurant Dashboard (B2B Web)

### 5.1 Авторизация

- Email + password (ресторанный аккаунт)
- Роли внутри ресторана: `Owner` / `Manager` / `Staff`
- Staff имеет доступ только к Inventory mode
- Manager — меню + инвентарь
- Owner — всё, включая биллинг и аналитику

### 5.2 Управление цифровым меню

- WYSIWYG-просмотр меню (рендер идентичен Guest App)
- Привязка Google Sheets URL → автоматическая синхронизация (существующая логика MVP)
- Ручная публикация изменений или auto-sync по расписанию
- Управление разделами (секции, табы, layout type)
- Загрузка логотипа, выбор темы (light/dark/custom accent)
- Мультиязычность: привязка дополнительных Google Sheets для других языков

### 5.3 Inventory Management

- Список всех позиций меню с текущим статусом
- Inline-редактирование статуса: `В наличии` / `Последние бутылки` / `Нет`
- Поле «количество бутылок» (опционально, только для внутреннего контроля)
- Batch update: выделить несколько позиций → изменить статус
- История изменений (кто, когда, что изменил) — для Manager/Owner
- Staff-режим: упрощённый UI только с инвентарём, без доступа к настройкам меню

### 5.4 Connector Module (Procurement Intelligence)

**Это центральная B2B функция платформы.**

**Поиск аналогов:**

```
Ресторан выбирает позицию из своего меню
  └→ Connector анализирует: регион, апелласьон, сорт, стиль, ценовой сегмент
       └→ Выдаёт список аналогов из каталога подключённых дистрибьюторов
            └→ Каждый аналог: фото, название, производитель, vintage, цена закупа
                 └→ Рядом — предложения от всех дистрибьюторов, у кого есть эта позиция
                      └→ Кнопка «Запросить предложение» или «Заказать»
```

**Сводка предложений (RFQ — Request for Quote):**

- Ресторан формирует список позиций для закупки
- Система автоматически отправляет запрос всем подключённым дистрибьюторам
- Дистрибьюторы отвечают ценой, наличием, условиями
- Ресторан видит сравнительную таблицу предложений
- Одним кликом: выбрать лучшее предложение → разместить заказ через платформу

**Алгоритм ранжирования аналогов:**
1. Релевантность (стиль, регион, ценовой сегмент) — базовый скоринг
2. Наличие на складе дистрибьютора
3. Featured Supplier (paid placement — прозрачный лейбл)
4. Рейтинг дистрибьютора (на основе выполненных заказов)

**История закупок:**
- Все B2B-заказы через платформу сохраняются
- График цен по позиции (историческая динамика)
- Топ-поставщики ресторана по объёму

### 5.5 Menu Performance Analytics

- Просмотры меню (по дням / неделям)
- Топ позиций по просмотрам (heatmap по секциям)
- Время сессии в меню
- Device split (iOS / Android / Desktop)
- Конверсия «просмотр → клик на позицию»
- Конверсия «клик на позицию → покупка онлайн» (для позиций с D2C)

### 5.6 Биллинг и подписка

- Текущий план и дата следующего списания
- История платежей
- Upgrade / Downgrade плана
- Управление способом оплаты

### 5.7 Планы подписки

## 6. Сегмент 3 — Distributor Portal (B2B Web)

### 6.1 Авторизация

- Email + password
- Роли: `Admin` / `Sales Manager` / `Warehouse`
- Warehouse — только обновление наличия
- Sales Manager — каталог + заказы
- Admin — всё, включая аналитику и биллинг

### 6.2 Управление каталогом

- Импорт позиций: CSV-загрузка (поля: SKU, название, производитель, регион, апелласьон, сорт, vintage, цена, фото URL, наличие)
- Ручное добавление / редактирование позиций
- Структура позиции:
  - Название (оригинал + русский)
  - Производитель
  - Страна / Регион / Апелласьон
  - Сорт (один или купаж)
  - Vintage (год)
  - Стиль (Red / White / Rosé / Sparkling / Dessert / Fortified)
  - Объём (750ml / 375ml / 1.5L / etc.)
  - Закупочная цена (B2B, видна только ресторанам)
  - Розничная цена (B2C, видна гостям)
  - Наличие на складе (количество или boolean)
  - Фото (ссылка или загрузка)
  - Дегустационные заметки (опционально)
  - Аллергены (SO2, etc.)
- Webhook / API-интеграция для синхронизации с внутренней WMS дистрибьютора (опционально)

### 6.3 Управление заказами

**B2C заказы (от гостей):**
- Входящие заказы с нотификацией (email + push)
- Подтверждение / отклонение заказа (таймаут: 2 часа)
- Передача в доставку: ввод трек-номера или интеграция с курьерским сервисом
- История завершённых заказов

**B2B заказы (от ресторанов через Connector):**
- Входящие RFQ (запросы на предложение) с деталями позиций
- Ответ на RFQ: цена + срок поставки + условия
- Подтверждение финального заказа от ресторана
- Invoicing (автоматическая генерация счёта через Hostera)

### 6.4 Зоны доставки и условия

- Настройка географии доставки (города / регионы)
- Минимальная сумма заказа (B2C и B2B — отдельно)
- Стоимость и сроки доставки по зонам
- Бесплатная доставка от суммы X (настраивается)

### 6.5 Priority Placement (Платное размещение)

- Покупка Featured Supplier статуса в Connector-поиске ресторанов
- Geo-таргетинг: конкретный город или рынок
- Дашборд показов и кликов по платному размещению
- Бюджет / период (недельные или месячные пакеты)

### 6.6 Allocation Marketing

- Создание промо-кампании для лимитированных позиций
- Аудитория: все Pass-подписчики / по городу / по вкусовому профилю
- Формат: push + in-app featured card в Guest App
- Аналитика кампании: охват, клики, конверсия в покупку

### 6.7 Аналитика (Analytics Tier)

- Объём продаж по позициям, регионам, стилям
- Популярность позиций в поиске Guest App (спрос без покупки)
- Что ищут рестораны в Connector (незакрытый спрос)
- Ценовые бенчмарки vs другие дистрибьюторы (анонимизировано)
- Сезонные тренды

### 6.8 Финансы и выплаты

- Баланс: начисленные выплаты за выполненные заказы
- Расписание выплат (например: каждые 7 дней)
- Детализация: каждый заказ, сумма, комиссия Hostera, итого к выплате
- История транзакций

## 7. Платёжная система

### 7.1 Hostera как агент

- Hostera принимает 100% платежа от покупателя (гость или ресторан)
- После подтверждения доставки/выполнения заказа — автоматическая выплата дистрибьютору за вычетом комиссии
- Период удержания: 2–5 рабочих дней (стандарт marketplace)

### 7.2 Комиссионная структура

| Тип транзакции | Комиссия Hostera |
|---|---|
| B2C (гость → дистрибьютор) | 8–12% от суммы заказа |
| B2B (ресторан → дистрибьютор через Connector) | 1.5–3% от суммы заказа |

### 7.3 Способы оплаты (Guest App)

- Банковская карта (Visa / Mastercard / Amex)
- Apple Pay
- Google Pay
- Сохранённая карта (для повторных заказов)

### 7.4 Способы оплаты (Restaurant Dashboard)

- Банковская карта (ежемесячный SaaS-платёж)
- ACH / банковский перевод (Enterprise)
- Автоматическое списание (subscription recurring)

## 9. Поиск: архитектурные требования

### 9.1 Глобальный поиск (Guest App)

- Поиск по полям: название, производитель, регион, апелласьон, сорт, vintage
- Полнотекстовый + fuzzy match (опечатки, транслит)

### 9.2 Connector Search (Restaurant Dashboard)

- Поиск по каталогу всех дистрибьюторов
- Similarity search по позиции меню (апелласьон + стиль + ценовой сегмент)
- Фильтр по: дистрибьютору / ценовому диапазону / наличию / сроку поставки
- Сортировка: Featured first → релевантность → цена
